# Animalese TTS 문제 해결 가이드

## MediaRecorder 0바이트 오류

### 증상
- WAV 변환 시 0바이트 파일 생성
- 콘솔에 "오디오 캡처에 실패했습니다 (0바이트)" 오류 메시지

### 해결 방법

#### 방법 1: 브라우저에서 테스트
```
1. Chrome 또는 Edge 브라우저 열기
2. 주소창에 chrome://flags 입력
3. "WebAudio" 검색
4. Experimental Web Platform features → Enabled
5. 브라우저 재시작
6. 앱 실행
```

#### 방법 2: Electron 실행 플래그
```bash
# Windows
./AnimaleseTyping.exe --enable-features=WebAudio

# 또는
./AnimaleseTyping.exe --enable-web-security=false
```

#### 방법 3: 앱 설정 수정
```
1. 앱을 우클릭 → "속성" → "호환성"
2. "고 DPI 설정 변경" → "감지 안 함"
3. "프로그램을 관리자 권한으로 실행" 체크
4. 확인 후 재시작
```

#### 방법 4: 오디오 드라이버 업데이트
```
1. 디바이스 관리자 열기
2. "오디오 입력 및 출력" 섹션 확인
3. 각 오디오 장치 → 우클릭 → "드라이버 업데이트"
4. "드라이버 자동 검색" 선택
```

## SRT 변환 실패

### 증상
- SRT 파일 로드 시 오류 발생
- Cue 변환 중 오류 메시지

### 해결 방법

#### 1. SRT 파일 형식 확인
```
올바른 형식:
1
00:00:01,000 --> 00:00:02,000
안녕하세요

2
00:00:02,500 --> 00:00:04,000
Hello World

잘못된 형식:
- 시간 형식이 다른 경우 (예: 00:00:01.000)
- 빈 줄이 없는 경우
- 인코딩이 UTF-8이 아닌 경우
```

#### 2. 파일 인코딩 확인
```
1. SRT 파일을 텍스트 에디터로 열기
2. "다른 이름으로 저장" → "인코딩: UTF-8" 선택
3. 저장 후 다시 시도
```

## 음성 프로필 저장 실패

### 증상
- Male/Female 버튼 작동 안함
- 음성 설정이 저장되지 않음

### 해결 방법
```
1. 앱 재시작
2. 설정 변경 후 즉시 앱 닫기
3. preferences.json 파일 삭제 (앱 다시 시작 시 자동 재생성)
   위치: %APPDATA%/animalese-typing/preferences.json
```

## 성능 문제

### 증상
- 변환 속도가 느림
- 앱이 느려짐 또는 응답 없음

### 해결 방법
```
1. 백그라운드 앱 최소화
2. 시스템 모니터 확인 (Ctrl+Shift+Esc)
3. 메모리 사용량이 1GB 이상이면 앱 재시작
4. 큰 텍스트는 여러 번에 나누어 변환
```

## 디버깅 모드 활성화

### 콘솔 로그 확인
```
1. F12 또는 Ctrl+Shift+I로 개발자 도구 열기
2. "Console" 탭 클릭
3. 오류 메시지 확인
4. "TTS 변환 시작", "녹음 시작" 등의 로그 확인
```

### 로그 레벨 설정
```
app.js 또는 설정 파일에서:
window.DEBUG_TTS = true;
```

## 알려진 제한사항

### 1. Electron 환경
- MediaRecorder API가 완전히 지원되지 않을 수 있음
- 브라우저에서보다 느릴 수 있음

### 2. 대용량 SRT 파일
- 1MB 이상 SRT 파일은 변환 시간이 오래 걸릴 수 있음
- 10,000자 이상 텍스트는 메모리 부족 발생할 수 있음

### 3. 멀티 모니터
- 듀얼 모니터 환경에서 앱 크기 변경 시 화면 밖으로 나가기 가능
- 해결: 앱 크기 수동 조정

## 자주 묻는 질문 (FAQ)

### Q: WAV 파일音质이 좋지 않아요.
**A**: 품질 설정을 "고품질"로 변경하세요.
- 저용량: 22kHz, 8bit (기본값)
- 표준: 44.1kHz, 16bit
- 고품질: 48kHz, 24bit

### Q: 변환된 WAV 파일을 재생할 수 없어요.
**A**: VLC Media Player 또는 Windows Media Player로 재생해보세요.
- 파일 형식이 올바르지 않을 수 있음
- 또는 코덱 문제가 있을 수 있음

### Q: 미리듣기가 작동하지 않아요.
**A**:
1. 시스템 볼륨 확인
2. 앱 볼륨 확인
3. 오디오 장치 확인 (헤드폰/스피커 연결 상태)

### Q: 한글이 올바르게 변환되지 않아요.
**A**:
1. 자모 분해 매핑 테이블 확인
2. 영어 알파벳으로만 변환됨 (한글 매핑이 누락된 경우)

### Q: 특수문자가 처리되지 않아요.
**A**: SFX 매핑 테이블 확인
- 공백, 마침표, 물음표, 느낌표 등
